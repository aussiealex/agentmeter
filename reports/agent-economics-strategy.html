AgentMeter

The missing economics layer for MCP agents.
Meter every tool call. Know what every agent task costs. Ship agents as products.

1. The One-Sentence Thesis

Agents are becoming products, but there's no Stripe for them. AgentMeter is an open-source MCP proxy that sits between agents and their tools, metering every interaction — so developers can track costs, set budgets, and eventually bill customers per task instead of per API key.

2. Why This, Why Now

The Problem

  • An agent calls 12 LLM endpoints, 5 MCP tools, and 3 APIs to complete one task
  • Nobody knows what that task cost
  • Nobody can set a budget ("don't spend more than $2 on this")
  • Nobody can bill a customer for it
  • Nobody can compare: "did the agent get cheaper or more expensive this week?"

Why MCP Is the Wedge

  • MCP is where agents connect to tools — it's the billing boundary
  • Adopted by Claude Code, Cursor, Windsurf, VS Code, OpenAI
  • Protocol is standardized — you can build middleware without vendor lock-in
  • Your MailSift MCP expertise is directly transferable
  • Nobody has built the metering layer for MCP yet

4. The 3-Phase Plan

Phase 1 The Proxy — Weeks 1-2

Ship the core: an MCP proxy that meters tool calls.

  • MCP proxy server in Python — receives tool calls, forwards to upstream MCP server, records everything
  • SQLite storage (you already know this pattern from MailSift)
  • CLI dashboard: agentmeter stats — show cost breakdown by tool, by session, by day
  • Config: point it at any MCP server via a simple JSON config (same format as Claude Code's MCP config)
  • Budget enforcement: set a max cost per session, proxy refuses calls when exceeded

Setup experience (the 60-second test):

# Install
pip install agentmeter

# Wrap any existing MCP server
agentmeter wrap --config mcp-servers.json

# Your Claude Code / Cursor now routes through AgentMeter
# Every tool call is metered automatically

# Check your costs
agentmeter stats --today
agentmeter stats --by-tool
agentmeter stats --by-session

What makes this shippable in 2 weeks:

  • MCP proxy is ~500 lines of Python (you've built MCP servers, a proxy is the same protocol both directions)
  • SQLite schema is a single table (you've done this in MailSift)
  • CLI stats is a few SQL aggregation queries + formatted output
  • No frontend needed — CLI-only for v1

Phase 2 The Dashboard — Weeks 3-4

Add visibility. Make the data useful beyond the CLI.

  • Local web dashboard (single HTML file served by the proxy, like MailSift reports)
  • Cost-per-task view: group tool calls into logical tasks, show total cost per task
  • Tool leaderboard: which MCP tools are called most, which cost most, which fail most
  • Trend charts: daily/weekly cost trends, usage patterns
  • Export: JSON/CSV export of all metering data
  • Session replay: see the full sequence of tool calls for any task

Open source launch:

  • GitHub repo with clean README, install instructions, screenshots
  • Post to Hacker News, r/LocalLLaMA, MCP Discord
  • Write a blog post: "I metered my Claude Code usage for a week — here's what I found"

Phase 3 The Economics Layer — Month 2+

Turn metering into monetization tooling for agent builders.

  • API keys with budgets: issue API keys to customers, each with a spend limit. Agent builders can give customers access to their MCP agent with built-in billing.
  • Usage reports per customer: "Customer X used 347 tool calls this month, estimated cost $12.40"
  • Webhook on budget events: notify when a customer hits 80% of budget, or when a session exceeds expected cost
  • Cost allocation rules: assign dollar costs to tool calls (e.g., "each database query costs $0.001, each LLM call costs $0.01")
  • Stripe integration: auto-generate invoices based on metered usage

This is where the paid tier emerges naturally: the hosted version of Phase 3 with team dashboards, Stripe integration, and SLAs.

8. Risks and Mitigations

RiskSeverityMitigation
MCP protocol changes break the proxy Medium MCP is under Linux Foundation now, versioned. Pin to stable spec. You'll be close to changes as an MCP builder.
Claude Code / Cursor add built-in cost tracking High They'll track their own costs, not cross-tool costs. AgentMeter sees ALL MCP servers, not just one host. Also: the economics layer (billing, budgets) isn't their business.
Langfuse adds MCP support Medium They'll add it as a feature, not a core primitive. Your advantage is being MCP-native and economics-focused, not observability-focused.
Nobody cares about agent costs yet Low Every developer who's run Claude Code for a day wants to know "what did that cost me?" The problem is already felt, just unsolved.
Solo dev burnout Medium Phase 1 is 2 weeks. If nobody cares after launch, you've lost 2 weeks, not 6 months. Fast validation.
